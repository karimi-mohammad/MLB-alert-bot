# مستند طراحی پروژه — سامانه اعلان مسابقات MLB در تلگرام

## هدف پروژه

طراحی یک سامانه خودکار برای اطلاع‌رسانی مسابقات **Major League Baseball (MLB)** در تلگرام که بدون نیاز به دخالت کاربر، برنامه مسابقات را دریافت کرده و در زمان‌های مشخص، اعلان‌های فارسی را در یک کانال تلگرام منتشر کند.

---

# اهداف اصلی

* دریافت خودکار برنامه تمامی مسابقات MLB
* ارسال اعلان در زمان‌های مشخص
* ارسال نتیجه پس از پایان مسابقه
* جلوگیری از ارسال پیام تکراری
* اجرای کاملاً خودکار
* بدون نیاز به سرور دائمی (استفاده از GitHub Actions یا سرویس مشابه)

---

# قابلیت‌های اصلی

## ۱. دریافت برنامه مسابقات

سیستم باید بتواند از API رسمی MLB اطلاعات زیر را دریافت کند:

* شناسه مسابقه (Game ID)
* تاریخ
* ساعت شروع
* تیم میزبان
* تیم مهمان
* وضعیت مسابقه
* زمان پایان
* نتیجه نهایی

---

## ۲. اعلان ۲۴ ساعت قبل مسابقه

در صورتی که تا شروع مسابقه تقریباً ۲۴ ساعت باقی مانده باشد، یک پیام در کانال منتشر شود.

نمونه محتوا:

* نام دو تیم
* تاریخ
* ساعت شروع (به وقت ایران)
* مدت زمان باقی‌مانده

---

## ۳. اعلان ۳ ساعت قبل مسابقه

سه ساعت مانده به شروع مسابقه، اعلان دوم ارسال شود.

هدف این اعلان یادآوری نزدیک بودن شروع مسابقه است.

---

## ۴. اعلان پایان مسابقه

پس از پایان مسابقه، نتیجه نهایی منتشر شود.

اطلاعات موردنیاز:

* تیم برنده
* تیم بازنده
* امتیاز هر دو تیم

نمونه:

```
🏁 پایان مسابقه

Dodgers 6
Yankees 4

🏆 برنده:
Los Angeles Dodgers
```

---

# قابلیت‌های اختیاری (قابل فعال یا غیرفعال)

سیستم باید امکان فعال یا غیرفعال کردن هر نوع اعلان را از طریق فایل تنظیمات داشته باشد.

مثال:

```
اعلان ۲۴ ساعت قبل      ✅
اعلان ۳ ساعت قبل       ✅
اعلان شروع مسابقه      ❌
اعلان پایان مسابقه     ✅
```

---

# جلوگیری از ارسال تکراری

سیستم باید اطمینان حاصل کند که هر اعلان فقط یک بار ارسال می‌شود.

برای هر مسابقه وضعیت‌های زیر ذخیره شوند:

* اعلان ۲۴ ساعت ارسال شده؟
* اعلان ۳ ساعت ارسال شده؟
* اعلان پایان ارسال شده؟

---

# الگوریتم کلی

```
اجرای Cron

↓

دریافت مسابقات امروز و فردا

↓

برای هر مسابقه

↓

اگر اعلان ۲۴ ساعت ارسال نشده

↓

اگر اختلاف زمان ≈ ۲۴ ساعت

↓

ارسال پیام

↓

ثبت وضعیت
```

همین الگوریتم برای اعلان ۳ ساعت و پایان مسابقه نیز تکرار می‌شود.

---

# دیتابیس

استفاده از SQLite برای ذخیره وضعیت اعلان‌ها.

نمونه جدول:

| فیلد       | توضیح               |
| ---------- | ------------------- |
| gameId     | شناسه مسابقه        |
| gameDate   | زمان شروع           |
| homeTeam   | تیم میزبان          |
| awayTeam   | تیم مهمان           |
| sent24h    | وضعیت اعلان ۲۴ ساعت |
| sent3h     | وضعیت اعلان ۳ ساعت  |
| sentFinal  | وضعیت اعلان پایان   |
| winner     | تیم برنده           |
| homeScore  | امتیاز میزبان       |
| awayScore  | امتیاز مهمان        |
| lastStatus | آخرین وضعیت مسابقه  |

# اجزای سیستم

## Scheduler

مسئول اجرای دوره‌ای سیستم.

پیشنهاد:

هر ۱۵ دقیقه یک بار اجرا شود.

---

## MLB API Client

مسئول دریافت اطلاعات مسابقات از API رسمی MLB.

وظایف:

* دریافت برنامه مسابقات
* دریافت وضعیت مسابقه
* دریافت نتیجه

---

## Parser

تبدیل اطلاعات خام API به ساختار داخلی پروژه.

---

## Database Layer

وظایف:

* ذخیره مسابقات
* ثبت اعلان‌های ارسال‌شده
* جلوگیری از ارسال تکراری

---

## Notification Engine

قلب پروژه.

وظایف:

* بررسی زمان مسابقات
* تصمیم‌گیری برای ارسال اعلان
* تولید متن پیام

---

## Telegram Sender

وظیفه:

ارسال پیام به کانال تلگرام.

# توسعه‌پذیری

معماری پروژه باید به گونه‌ای باشد که در آینده بتوان بدون تغییر بخش‌های اصلی، لیگ‌های دیگر را نیز اضافه کرد.

برای مثال:

* NBA
* NFL
* NHL
* MLS
* Premier League
* La Liga

تنها با اضافه کردن یک ماژول جدید برای دریافت اطلاعات مسابقات.

---

# پیشنهادهای توسعه آینده

* اعلان شروع مسابقه
* خلاصه مسابقات روز
* اعلام برنامه مسابقات فردا
* انتخاب تیم‌های مورد علاقه
* ارسال تصویر لوگوی تیم‌ها
* استفاده از قالب‌های زیباتر برای پیام‌ها
* پشتیبانی از چند کانال
* پنل مدیریتی تحت وب
* داشبورد مشاهده لاگ‌ها
* ذخیره آمار مسابقات برای گزارش‌گیری

# نتیجه نهایی

سامانه‌ای سبک، رایگان، خودکار و قابل توسعه که بدون نیاز به سرور دائمی، مسابقات MLB را پایش کرده و اعلان‌های زمان‌بندی‌شده و نتایج نهایی را به‌صورت فارسی در کانال تلگرام منتشر می‌کند. طراحی ماژولار آن امکان افزودن لیگ‌های ورزشی دیگر و قابلیت‌های جدید را در آینده بدون بازنویسی بخش‌های اصلی فراهم می‌کند.
